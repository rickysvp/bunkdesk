Walk-in 表单布局：3 个方案

在现有 8 字段基础上扩展到 ~13 字段。下方 3 个 mockup 展示了不同的组织方式

A

线性增长 - 现有布局直接加字段

保持 2 段（个人信息 + 入住信息），把所有新字段平铺进去。最少改动，但表单高度从 380px 涨到 ~620px，需要向下滚动。

PROS · 改动最小、字段全可见
CONS · 表单变长、滚动多

B

分块折叠 - 3 个可折叠 Section

分成 Personal / Stay / ID & Source 三个折叠区，每段带图标 + 摘要。必填字段在 Personal 段，其余默认折叠可展开。最适合 "更利于操作"。

C

左右两栏 - 主信息 + 摘要栏

左侧 60% 显示所有输入字段，右侧 40% 实时显示「摘要卡」（已填项汇总 + 还差什么）。参考截图接近这个模式，但更紧凑。
