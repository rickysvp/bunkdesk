详情卡布局：3 个方案

新字段（Email/Phone/ID Type/Arrival Time/Referral）要插到现有 4 张卡的哪里？

A

线性扩展 - 新字段塞进现有卡

Header 升级为「客户摘要卡」（含 Email/Phone/Arrival Time），Verification 卡增加 ID Type 显示，Notes 卡增加 Referral 字段。改动最小，结构不动。

Detail View - Linear Expansion
John Smith · USA
📧 john@mail.com
📞 +1-555-0100
🕒 Arriving ~3pm
Verification
✓ Passport P1234567 scanned
Payment
⚠ Unpaid $255
Notes
Referral: Booking.com
⚡ Auto Assign & Check-in

PROS · 改动最小、与 Walk-in 风格统一
CONS · Header 变拥挤

B

新增 Contact 卡 - 5 张卡结构

在 Header 下面插入新的「Contact」卡（独占显示 Email/Phone/Arrival Time/Referral），Header 恢复简洁。Verification 独立并加 ID Type。

USA · 3 nights · check-out 6/18

📧 john@mail.com
📞 +1-555-0100
🕒 Arriving ~3pm
🔗 Referral: Booking.com
Verification
✓ Passport P1234567
Payment
⚠ Unpaid $255
Notes: Prefers bottom bunk

PROS · 信息分组清晰、Header 简洁
CONS · 多 1 张卡、整体变长

C

两栏布局 - 信息 + 操作

左 60% 显示客户信息（Header + Contact + ID 合并为一张信息卡），右 40% 显示操作（Verification/Payment/Notes/Bed Assignment 整合为操作面板）。

PROS · 信息操作分离清晰
CONS · 右栏窄、详情卡需更紧凑
