① 旧数据迁移（localStorage 启动时）

src/utils/guestMigration.ts · 新增
// 启动时调用，对每个 Guest 做一次性兼容
export function migrateGuest(g: Guest): Guest {
const result = { ...g };
// 1. 姓名拆分（仅当 firstName/lastName 缺失时）
if (!g.firstName && !g.lastName && g.name) {
const parts = g.name.trim().split(/\s+/);
result.firstName = parts[0] || '';
result.lastName = parts.slice(1).join(' ') || '';
}
// 2. IDType 默认
if (!g.idType) result.idType = 'passport';
// 3. 其它新字段保持 undefined（详情卡显示 "—"）
return result;
}

关键策略

  • 启动时一次性遍历 + 写回 localStorage
  • name 字段保留（向后兼容）
  • idType 默认 passport
  • 邮箱/电话/arrivalTime/referral 为 undefined，详情卡显示"未填写"+提示

注意点

  • 多字姓名（西班牙/中国）按首空格拆可能不完美，但 iCal 也同样限制
  • 版本号 v1.6.0 写入 localStorage 检测是否需要迁移

④ iCal 导入智能解析（src/components/ICalImport.tsx）

SUMMARY 解析逻辑
function parseSummaryToName(summary: string): {first: string, last: string} {
const s = summary.trim();
// 1. "Last, First" 格式
if (s.includes(',')) {
const [last, first] = s.split(',').map(x => x.trim());
return { first, last };
}
// 2. "First Last" 格式（按首空格拆）
const parts = s.split(/\s+/);
if (parts.length === 1) return { first: parts[0], last: '' };
return { first: parts[0], last: parts.slice(1).join(' ') };
}
