# 深度业务逻辑与交互完整性优化方案

## 摘要

解决两个核心问题：
1. **Dashboard 概览缺少性别分布和房间维度数据** — 无法一眼看出男女入住情况和各房间入住率
2. **Shift Log 缺少来源追踪** — 不知道日志是怎么产生的（手动创建？系统自动？从哪个操作触发？）

同时提升整体交互完整性：操作联动、数据闭环、状态同步。

  const empty = room.beds.filter(b => b.status === 'empty').length;

## 假设与决策

1. **自动日志不阻塞操作** — 自动添加日志是副作用，不应影响主操作的成功/失败
2. **自动日志默认已解决** — 系统自动生成的日志（如退房记录）默认 `isResolved: true`，仅作为记录
3. **手动日志默认未解决** — 人工创建的日志默认 `isResolved: false`，需要手动标记解决
4. **来源标签用颜色区分** — 与现有分类颜色体系一致，不引入新颜色
5. **房间入住率显示性别小计** — 用 ♂♀ 符号 + 数字，不占太多空间
6. **Dashboard 点击跳转用 setActiveTab** — 与现有导航机制一致
