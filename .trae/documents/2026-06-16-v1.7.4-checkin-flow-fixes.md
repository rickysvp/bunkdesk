# 前台入住交互修复 v1.7.4

> **Plan Mode** — 待用户批准后实施

## 1. Summary

修复前台入住流程的 4 个核心交互 bug：

1. **Walk-in 表单 gender 非必填** — 客人没选性别也能创建，但床位分配引擎需要 gender
2. **Auto-Assign 不要求先收款** — 客人欠款状态下也能直接入住，应该前置必须先收款
3. **性别与房间偏好不联动** — 选了 Male+Female Dorm、Female+Male-only 都能保存，应该联动过滤
4. **床位卡显示/交互优化** — 重新设计分配卡，包含更清晰的信息层级、状态指示和选中态

## 2. Current State Analysis

- 文件：`src/components/CheckInPanel.tsx:132`

- 现状：直接调 `autoAssignBed` 然后 `setCheckInSuccess`
- 缺失：paymentStatus==='unpaid' 时直接 disable 按钮 + 提示

- 文件：`src/components/CheckInPanel.tsx:119-128` (handleCheckIn)
- 现状：手动选床位 + 手动点 Complete Check-in 时也直接 `assignArrival`
- 缺失：paymentStatus==='unpaid' 时应要求先付款

### Bug 3: 性别与房间偏好不联动
- 文件：`src/components/CheckInPanel.tsx:347-350` (Walk-in roomPreference Select)
- 现状：无论 gender 如何都显示 `dorm-mixed/dorm-female/private`
- 应该：
  - gender=male → 不显示 `dorm-female`（男不能进女宿）
  - gender=female → 三个都可以，但 `dorm-female` 排序在前
  - gender=other/undefined → 三个都显示
- bedPreference 也要联动：房间已满上铺才让选 top

### Bug 4: 床位卡显示/交互问题
- 文件：`src/components/CheckInPanel.tsx:597-647` (床位卡列表)
- 现状：2 列 6 张卡，min-h-140px，每张信息密集但视觉混乱
- 应该：
  - **选中态用大蓝边 + ring** 更突出
  - **不适用的床位变灰**（如 male 客人 + female dorm 不应出现）
  - **价格对比更清晰**（单价、总价、差额分三行）
  - **推荐原因放最下面**，不被挤掉

### File 2: `src/utils/bedAllocator.ts`

#### 2a. 女性 + mixed dorm 不应得 genderMatch tag
虽然技术上合规，但视觉上 female 客人看到 female dorm 才是真正的"性别匹配"。把 line 132-137 改为：

```ts
// Female in mixed dorm: acceptable but no gender match tag
// (genderMatch will be set in dorm-female case above)
```

这样 `genderMatch` 只在 `dorm-female && female` 或 `dorm-mixed && male` 时为 true，更准确。

#### 2b. 增强 hard-block
- male + dorm-female → null（已有）
- female + dorm-male → null（新增，需要 types.ts 引入 dorm-male）

实际产品里没有 dorm-male 房间，所以保留现状即可。

## 4. Assumptions & Decisions

- **未付款时 Auto-Assign 完全禁用**（不只是弹窗），更清晰
- **Complete Check-in 同样禁用**，因为手动选床位也涉及入住
- **gender 联动**：female 看不到 male dorm（不存在），但 female+female 优先；male+female 物理上 hard block（已实现）
- **价格显示**：单价用 `text-xl`，比标签字号更大，视觉聚焦
- **选中态**：用 `scale-[1.02] + ring-4 + shadow-lg` 提升视觉
