# 前台入住床位卡 + 价格同步修复 v1.7.2

> **Plan Mode** — 待用户批准后实施

## 1. Summary

修复前台入住「待办理卡片」的两大类问题：

**A. 数据 bug**：选不同床位时，客人 `totalAmount` 不会跟着变（一直用 Walk-in 时算的 `nights * AVG_PRICE`）。必须从 `assignArrival` / `autoAssignBed` 中按所选床位重新计算总价。

**B. 视觉体验差**（截图确认）：
- 床位卡 6 个排成 3×2，但每张卡只显示房型+床名+价格+标签，**没有价格对比、总价、推荐理由说明**，用户根本不知道为什么是「BEST」
- "Gender/Fill/Low frag" 三个灰色小标签重复出现，没有突出**核心推荐原因**
- 标签颜色不鲜明（♀ Gender 是蓝、Fill 是灰、Low frag 是橙）信息层级混乱
- 床位卡之间没有价格高低的视觉对比（女生多人间 ¥95/¥105，混合 ¥85/¥95）
- 整体信息密度低，浪费了大量空间

## 2. Current State Analysis

### Bug 根因
- `src/HostelContext.tsx:342-369` `assignArrival(guestId, bedId)` 直接把 `assignedGuest` 拷贝到 bed.guest，**没有更新 `totalAmount`**
- `src/HostelContext.tsx:371-381` `autoAssignBed` 调用 `assignArrival` 同样问题
- `src/components/CheckInPanel.tsx:119-123` `handleCheckIn` 也只是调 `assignArrival`
- `src/components/CheckInPanel.tsx:453` 详情卡 `paymentDue` 显示的是 `selectedGuest.totalAmount ?? (nights * AVG_PRICE)`，所以永远不变
- `src/components/CheckInPanel.tsx:603` 床位卡只显示单晚价格（`formatCurrency(score.pricePerNight, language)`），不是总价

### 视觉问题截图分析
当前床位卡布局（截图位置 `CheckInPanel.tsx:589-617`）：
- 网格 `grid-cols-2 gap-1.5 max-h-72`，每张卡 `p-2.5 min-h-[60px]`
- 显示内容：房型（大字）+ 床名·房号·价格 + 3-4 个标签
- 缺失：**总价**、**与当前均价差额**、**推荐原因简述**

## 3. Proposed Changes

### File 1: `src/HostelContext.tsx`

**修改 `assignArrival` (line 342-369)**：分配床位时根据所选床位重新计算总价

```tsx
const assignArrival = useCallback((guestId: string, bedId: string) => {
  const guest = arrivals.find((g) => g.id === guestId);
  
  // Find room/bed to get the actual price
  const room = rooms.find((r) => r.beds.some((b) => b.id === bedId));
  const bed = room?.beds.find((b) => b.id === bedId);
  const actualPricePerNight = room && bed
    ? (room.pricePerNight || 0) + (bed.bedType === 'bottom' ? (room.bottomBunkPremium || 0) : 0)
    : 0;
  
  setArrivals((prev) => prev.filter((g) => g.id !== guestId));
  setRooms((prevRooms) =>
    prevRooms.map((r) => ({
      ...r,
      beds: r.beds.map((b) => {
        if (b.id === bedId) {
          const assignedGuest = prevRooms
            .flatMap((r2) => r2.beds.map((b2) => b2.guest))
            .find((g) => g?.id === guestId) || guest;
          if (assignedGuest) {
            // Recalculate totalAmount based on the actual bed's price
            const updatedGuest = {
              ...assignedGuest,
              totalAmount: Math.round(actualPricePerNight * (assignedGuest.nights || 1) * 100) / 100,
            };
            return { ...b, status: "occupied", guest: updatedGuest };
          }
        }
        return b;
      }),
    })),
  );
  
  if (guest) {
    addAutoNote(`${guest.name} checked in`, "checkin", guestId, "guest");
    logAction(guestId, 'check-in', `Checked in → ${room?.name} - ${bed?.name} (${formatCurrency(actualPricePerNight, currentLanguage())}/night)`);
  }
}, [arrivals, rooms, addAutoNote, logAction]);
```

### File 3: `src/components/CheckInPanel.tsx` — Auto-Assign 后总价同步

`assignArrival` 已经更新了 totalAmount，但 CheckInPanel 的 `selectedGuest` 是从 `arrivals` 来的（已过滤），需要从 `bed.guest` 重新读取。`handleCheckIn` 调用 `assignArrival` 后会用 `bed.guest`，但 detail view 仍指向 `arrivals` 的 `selectedGuest`。

修复：在 `handleCheckIn` 后从房间中刷新 selectedGuest（line 119-123）

```tsx
const handleCheckIn = () => {
  if (selectedGuestId && selectedBedId) {
    assignArrival(selectedGuestId, selectedBedId);
    // Find the assigned guest in the new room
    const newRoom = rooms.find(r => r.beds.some(b => b.id === selectedBedId));
    const newBed = newRoom?.beds.find(b => b.id === selectedBedId);
    if (newBed?.guest) {
      setSelectedGuestId(null);
      setSelectedBedId(null);
    }
  }
};
```

实际上，因为 `selectedGuestId` 在 `assignArrival` 后会从 arrivals 中消失（被移到 bed.guest），UI 会自动重置。我们需要的是 `setSelectedGuestId` 不重置，detail view 改为同时查找 `arrivals` 和 `rooms[].beds[].guest` 来获取最新的 totalAmount。

更简单的方法：在 `setSelectedGuestId` 后从 `rooms` 找到这个 guest 的最新数据。

## 4. Assumptions & Decisions

- **`totalAmount` 重新计算时机**：在 `assignArrival` 中按所选床位重算，保留 `paidAmount`（不重置已付金额）
- **价格视觉对比**：用与 `AVG_PRICE` 的差值，正数琥珀色、负数绿色
- **床位卡高度**：从 `min-h-[60px]` 增大到 `min-h-[140px]`，每张卡内容更丰富
- **BEST 标签位置**：从 `-top-1.5 left-2` 改为 `-top-2 left-3`，更突出
- **床位卡选中态**：蓝色 ring（`ring-2 ring-blue-200`）替代之前的全黑底白字，更柔和
