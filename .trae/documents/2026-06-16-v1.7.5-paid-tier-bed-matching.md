# 按支付金额匹配床位推荐 v1.7.5

> **Plan Mode** — 待用户批准后实施

## 1. Summary

修复床位推荐引擎的核心逻辑缺失：**根据客人实际支付的金额推荐对应价位的床位**。

- 客人付了 top bunk 的钱（`totalAmount = nights * topPrice`）→ 优先推荐 top bunk
- 客人付了 bottom bunk 的钱（含 premium）→ 优先推荐 bottom bunk

之前 `bedAllocator` 完全没有考虑 `bedPreference` 和 `totalAmount`，导致**付了 bottom bunk 钱的人被推荐 top bunk 床**这种严重错误。

## 2. Current State Analysis

### Bug 根因
- 文件：`src/utils/bedAllocator.ts:96-158` `computeBedScore`
- 现状：评分只考虑 fillExisting (40) + genderMatch (30) + preferenceMatch (20) + fragmentation (10)
- **缺失**：
  - 读 `bed.bedType` 和 `guest.bedPreference`（"上下铺喜好"已收集但未使用）
  - 读 `guest.totalAmount` / `paidAmount` 来判断"客人付的是哪档的钱"

### 数据流
1. Walk-in 表单已收集 `bedPreference: 'top' | 'bottom' | 'any'`
2. 创建客人时设了 `totalAmount = nights * AVG_PRICE`（来自 v1.7.0 的 AVG_PRICE）
3. `assignArrival` (v1.7.2) 会按所选床位**重新计算** `totalAmount`
4. **问题**：在推荐阶段，scoredBeds 不知道客人付了多少钱该匹配哪档

## 3. Proposed Changes

### File 1: `src/utils/bedAllocator.ts`

#### 1a. 从 `guest.totalAmount` 反推"应该住哪档"
```ts
// In computeBedScore, add new logic before scoring:
// Detect which tier the guest has paid for based on their totalAmount
// vs the room's price range (top vs bottom).

const topPrice = room.pricePerNight;  // Top bunk = base price
const bottomPrice = room.pricePerNight + (room.bottomBunkPremium || 0);  // Bottom = +premium
const expectedTotalForTop = topPrice * (guest.nights || 1);
const expectedTotalForBottom = bottomPrice * (guest.nights || 1);
const actualTotal = guest.totalAmount || 0;

// Tolerance: ±5% to handle rounding
const tolerance = 0.05;
const paidForBottom = actualTotal > 0 && 
  Math.abs(actualTotal - expectedTotalForBottom) / expectedTotalForBottom < tolerance;
const paidForTop = actualTotal > 0 && 
  Math.abs(actualTotal - expectedTotalForTop) / expectedTotalForTop < tolerance;
```

#### 1b. 添加 bed-tier match 评分
```ts
// ── 5. Bed tier match (weight 50 - high priority) ──
// If guest paid for top, prefer top bed. If paid for bottom, prefer bottom.
const WEIGHT_BED_TIER = 50;
let bedTierMatch = false;

if (paidForBottom && bed.bedType === 'bottom') {
  score += WEIGHT_BED_TIER;
  bedTierMatch = true;
  reasons.push('tier-match-bottom-paid');
} else if (paidForTop && bed.bedType === 'top') {
  score += WEIGHT_BED_TIER;
  bedTierMatch = true;
  reasons.push('tier-match-top-paid');
} else if (guest.bedPreference === 'bottom' && bed.bedType === 'bottom') {
  // User explicitly chose bottom preference
  score += Math.floor(WEIGHT_BED_TIER * 0.8);
  bedTierMatch = true;
  reasons.push('tier-pref-bottom');
} else if (guest.bedPreference === 'top' && bed.bedType === 'top') {
  score += Math.floor(WEIGHT_BED_TIER * 0.8);
  bedTierMatch = true;
  reasons.push('tier-pref-top');
}
```

权重 50 高于 fillExisting (40)，确保**按支付金额匹配的床位总是排第一**。

#### 1c. 添加 bedTierMatch 到 BedScore 接口

## 4. Assumptions & Decisions

- **付费金额容差 ±5%**：处理 round 误差
- **bedPreference 优先级低于 paidAmount**：用户付了某档的钱比"我说想住哪"更刚性
- **bottom bunk 永远更贵**（来自 `bottomBunkPremium`），没有反例
- **totalAmount 为 0 时**（未填金额）→ 走 bedPreference 逻辑
- **bedTierMatch 权重 50 最高**，因为这是用户实际付钱的指示
