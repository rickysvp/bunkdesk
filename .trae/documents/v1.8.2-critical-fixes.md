# v1.8.2 关键问题止血修复

## Summary

基于 BMAD-METHOD 深度代码审查发现的 5 个 Critical 问题和若干 High 问题，进行第一批止血修复。目标：消除用户可见缺陷、防止数据错误、建立基础健壮性。不包含大规模重构。

## Current State Analysis

### 问题 1: i18n `t() || 'fallback'` 系统性失效
- `i18nContext.tsx:2212` — `t()` 缺 key 时返回 `path` 字符串（truthy），导致 `|| 'fallback'` 永不触发
- 全项目 **216 处** `t('key') || 'fallback'` 模式分布在 15 个文件
- 4 个确认缺失的 key：`common.cancel`、`common.save`、`checkin.collectBeforeAssign`(en)、`checkin.noAvailableBeds`(en+zh)
- 用户在 UI 中会看到 `"common.cancel"` 等原始 key 字符串

### 问题 2: 无 Error Boundary
- `main.tsx` 直接渲染 `<App />`，无错误边界
- 任何子组件抛错导致整页白屏，用户只能刷新

### 问题 3: settlePayment 不校验金额
- `HostelContext.tsx:522-536` — 无条件设 `paymentStatus: "paid"`，即使 `paidAmount < totalAmount`
- 不更新 `paidAmount`、不写审计日志、不同步 `bed.reservations`

### 问题 4: 床位分配未拦截日期冲突
- `bedAllocator.ts:46` — 只跳过 `occupied`/`cleaning`，`reserved` 床位仍参与评分
- 客人入住日期与现有预订重叠时仍被推荐，导致超售

### 问题 5: addArrival / importArrivals 审计日志丢失
- `HostelContext.tsx:555-560` — `logAction` 在 `arrivals` 闭包中查找新客人，但 `setArrivals` 异步，找不到直接 return
- `HostelContext.tsx:584-590` — `importArrivals` 同样问题
- `logAction` 实现（228-258 行）：`arrivals.find(g => g.id === guestId)` 用的是旧闭包

## Proposed Changes

### 1. i18n 修复 — `src/i18nContext.tsx`

**1a. 修改 `t()` 缺 key 行为（行 2210-2212）**

当前：
```ts
if (raw === undefined) {
  return options?.defaultValue ?? path;
}
```
改为：
```ts
if (raw === undefined) {
  return options?.defaultValue ?? '';
}
```
**Why**: 返回空字符串（falsy）后，216 处 `t('key') || 'fallback'` 自动生效。比返回 key 路径字符串对用户更友好（看到英文 fallback 比看到 `common.cancel` 好）。

**1b. 补全缺失 key**

在 `common` 命名空间（en 和 zh）添加：
- `cancel: "Cancel"` / `"取消"`
- `save: "Save"` / `"保存"`

在 `checkin` 命名空间（en）添加：
- `collectBeforeAssign: "Please collect payment before assigning bed"`

在 `checkin` 命名空间（en + zh）添加：
- `noAvailableBeds: "No available beds"` / `"没有可用床位"`

### 2. Error Boundary — 新建 `src/components/ErrorBoundary.tsx`

创建 class 组件 ErrorBoundary：
- `static getDerivedStateFromError` 捕获错误
- `componentDidCatch` 记录错误到 console
- Fallback UI：显示错误信息 + "刷新页面"按钮
- 在 `main.tsx` 中包裹 `<App />`

**修改 `src/main.tsx`**：
```tsx
<StrictMode>
  <ErrorBoundary>
    <App />
  </ErrorBoundary>
</StrictMode>
```

### 3. settlePayment 金额校验 — `src/HostelContext.tsx:522-536`

当前：
```ts
const settlePayment = useCallback((guestId: string) => {
  setArrivals((prev) => prev.map((g) => (g.id === guestId ? { ...g, paymentStatus: "paid" } : g)));
  setRooms(/* 同步 */);
}, []);
```

改为：
```ts
const settlePayment = useCallback((guestId: string) => {
  setArrivals((prev) => prev.map((g) => {
    if (g.id !== guestId) return g;
    const newPaidAmount = g.totalAmount || 0;
    const newStatus = newPaidAmount >= (g.totalAmount || 0) ? "paid" : "partial";
    return { ...g, paidAmount: newPaidAmount, paymentStatus: newStatus };
  }));
  setRooms((prevRooms) => prevRooms.map((r) => ({
    ...r,
    beds: r.beds.map((b) => {
      if (b.guest?.id !== guestId) return b;
      const g = b.guest;
      const newPaidAmount = g.totalAmount || 0;
      const newStatus = newPaidAmount >= (g.totalAmount || 0) ? "paid" : "partial";
      return { ...b, guest: { ...g, paidAmount: newPaidAmount, paymentStatus: newStatus } };
    }),
  })));
  logAction(guestId, 'payment', `Payment settled: ${formatCurrency(0, currentLanguage())}`, { amount: 0 });
}, [logAction]);
```

**Why**: settlePayment 的语义是"结清余额"，应将 `paidAmount` 设为 `totalAmount`，然后标记 `paid`。补审计日志。

### 4. 床位日期冲突拦截 — `src/utils/bedAllocator.ts:43-46`

当前：
```ts
if (bed.status === 'occupied' || bed.status === 'cleaning') continue;
```

改为（在 continue 后添加日期冲突检查）：
```ts
if (bed.status === 'occupied' || bed.status === 'cleaning') continue;

// 检查日期冲突：客人入住日期与现有预订重叠则跳过
if (bed.reservations && bed.reservations.length > 0) {
  const guestCheckIn = parseISO(guest.checkInDate);
  const guestCheckOut = parseISO(guest.checkOutDate);
  const hasConflict = bed.reservations.some(res => {
    const resCheckIn = parseISO(res.checkInDate);
    const resCheckOut = parseISO(res.checkOutDate);
    return guestCheckIn < resCheckOut && resCheckIn < guestCheckOut;
  });
  if (hasConflict) continue;
}
```

**Why**: 日期区间重叠判断 `[guestCheckIn, guestCheckOut)` ∩ `[resCheckIn, resCheckOut)` ≠ ∅。需确保 `parseISO` 已 import（行 202 已有）。

### 5. 审计日志修复 — `src/HostelContext.tsx`

**5a. 修改 `logAction` 签名（行 228-258）**

const logAction = useCallback((
  guestId: string,
  type: GuestLogType,
  description: string,
  opts?: { amount?: number; meta?: Record<string, any>; guestSnapshot?: Guest },
) => {
  // 优先使用传入的 guest 快照，避免闭包陈旧问题
  let guest: Guest | undefined = opts?.guestSnapshot;
  if (!guest) {
    guest = arrivals.find((g) => g.id === guestId);
  }
  if (!guest) {
    for (const r of rooms) { /* ... 现有 rooms 查找逻辑 ... */ }
  }
  if (!guest) return;
  addGuestLog({ /* ... 现有逻辑 ... */ });
}, [arrivals, rooms, addGuestLog]);
```

logAction(newGuest.id, 'created', `...`, { guestSnapshot: newGuest });
```

**5c. 修改 `importArrivals`（行 584-590）**

for (const g of newGuests) {

## Assumptions & Decisions

1. **strict 模式不在此批** — 开启后可能产生大量编译错误，不适合"止血"定位，放到第二批
2. **`t()` 返回空字符串而非 key 路径** — 这是刻意决策，让 216 处 `|| 'fallback'` 自动生效，避免逐个修改
3. **settlePayment 语义为"结清"** — 将 `paidAmount` 设为 `totalAmount`，而非简单标记 `paid`
4. **Error Boundary 用 class 组件** — React 仍需 class 组件实现 error boundary，这是官方 API
5. **日期冲突用区间重叠判断** — `guestCheckIn < resCheckOut && resCheckIn < guestCheckOut`，半开区间
